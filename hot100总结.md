## 283.移动零

![image-20260301213915504](./images/image-20260301213915504.png)

题目要求原地修改数组，满足将0全部放在末尾，而非零数字的相对顺序不变。

这里使用双指针：

指针1为待填充非零数字的索引指针，指针2为遍历中发现非零数字的索引指针

指针2从前往后遍历数组，当发现非零元素时，则与指针1交换元素，指针1初始为0（即数组开始），在与指针2交换完毕元素后，则向前移动一位。这样当指针2遍历完毕数组时，指针1及其前的元素便是保持相对顺序的非零数字。



## 11.盛水最多的容器

![image-20260302135208290](./images/image-20260302135208290.png)

理论上可以通过**双循环**来解决，时间复杂度是 o(n2)：`for i in range(n-1): for j in range(i+1,n):`

相当于是n*n的矩形，保证i<j，则**搜索空间是矩形的一半，即不含对角线的三角形**。

此处使用**双指针来优化时间复杂度，缩小搜索空间**：

left,right分别从头和尾开始，left只能向右移动，right只能向左移动，二者不能重合。

- 假设left<right，则移动right将毫无收益（短板效应），盛水量只会更少，那么我们就将right处于当前位置（left处于任意<right的位置）的这些组合剪枝掉。
- 假设left>right，则移动left将毫无收益（短板效应），盛水量只会更少，那么就将left处于当前位置（right处于任意>left的位置）的这些组合剪枝掉。

这样就**将三角形的搜索空间缩小为线性搜索空间**，时间复杂度 o(n2) -> o(n)：

- 当left<right，则移动left，计算此时盛水量并与历史最大值比较；
- 当right<left，则移动right，计算此时盛水量并与历史最大值比较。



## 15.三数之和

![image-20260302140659792](./images/image-20260302140659792.png)

先不管重复的问题，聚焦**查全**的问题。题目要求i,j,k三者应是不同索引，则通过暴力的**三重循环**可以遍历所有可能：

`for i in range(n-2): for j in range(i+1,n-1): for k in range(j+1,n):`

上述通过要求 i<j<k，来实现在三者不相同的情况下遍历所有可能，时间复杂度为 o(n3)，显然过高，故使用**双指针来优化 j、k 的两个循环的复杂度**。

单独看 j、k的双重循环，本身时间复杂度是 o(n2)，我们先用 num.sort() 来通过 o(nlogn) 的时间复杂度进行排序。

之后让 j = i+1，k = n，其中，j只能向右移动，k只能向左移动，要求 j<k：

- 若 nums[i]+nums[j]+nums[k] == 0，则记录当前答案，j++，k--；
- 若 nums[i]+nums[j]+nums[k] > 0，则 k--，将整体和变小，看之后是否能到0；
- 若 nums[i]+nums[j]+nums[k] < 0，则 j++，将整体和变大，看之后是否能到0。

这样i的循环是 o(n)，j与k双指针是 o(n)，整体时间复杂度从 o(n3) 降到了 o(n2)。

**剪枝**：若nums[i]>0，因为nums[j]和nums[k]只会大于等于nums[i]，则三者和不可能为0，直接continue到下一个i即可。

上述可以解决将全部合理答案找到的问题，但是由于**数组中存在重复元素**，可能**导致**所找到的**三元组存在重复**，下面解决**去重**问题：

在排序后，数组的重复元素一定会相邻，我们只要**保证不让i、j、k使用重复元素**，所以在移动i、j、k的时候要检查移动后的数值是否与移动前一样，如果一样则继续移动，在具体代码里会存在两处：

- 外层i的循环，当i>0时（即 i-1 为有效索引），检查 nums[i] 与 nums[i-1] 是否相同，若相同则continue；
- 内层jk双指针，当 nums[i]+nums[j]+nums[k] == 0 时记录答案，此时j++和k--要保证移动后不能与记录答案时的 nums[j]、nums[k] 数值相同（当不记录答案，即nums[i]+nums[j]+nums[k] != 0 时，则无所谓）。



## 42.接雨水

![image-20260302143252000](./images/image-20260302143252000.png)

先将题目建模：对于某个柱子，其所能接雨水的量取决于它左侧的最高柱、右侧的最高柱以及自身高度，具体接水量是 **min(左侧最高，右侧最高)-自身高度**。

在计算最终接水量时，遍历每个柱子，只要知道其左右两侧最高的高度，便可以计算当前柱子的接水量。

### 方法1：前后缀数组

我们建立与 height 同长的两个数组 pre_max 和 suf_max：

- pre_max 记录**当前及其左侧范围内柱子的最大高度**
- suf_max 记录**当前及其右侧范围内柱子的最大高度**

![621517ff-2037-490b-9529-7cd6fb4bada5](./images/621517ff-2037-490b-9529-7cd6fb4bada5.png)

然后遍历每个柱子，按照 **min(左侧最高，右侧最高)-自身高度** 便可计算当前柱的接水量。

本方法的时间复杂度为 o(n)，空间复杂度为 o(n)。

### 方法2：双指针

双指针方法旨在将空间复杂度降低为 o(1)。

让 left=0, right = n-1，left只能向右移动，right只能向左移动，保证left<=right。此时 pre_max 和 suf_max 为数值：

- pre_max 记录**left及其左侧范围内柱子的最大高度**
- suf_max 记录**right及其右侧范围内柱子的最大高度**

此时，如果 pre_max<suf_max，则当前位置的短板就是pre_max，其接水量即 **pre_max - 当前高度**，计算完接水量后将 left 向右移动一位，同时更新pre_max。这样来理解：当前位置的左侧最大高度是pre_max，右侧最大高度大于等于suf_max（right经过的右半块的最高是suf_max，right向左直到left之间仍然可能存在柱子高度超过suf_max），则当前位置的短板一定是pre_max。若 suf_max<pre_max 同理可得。

## 239.滑动窗口最大值

![image-20260304151946915](./images/image-20260304151946915.png)

遍历滑动窗口你肯定会，但是在窗口内找最大值，若是用max(数组)，则时间复杂度会超标，所以重点在于**以o(1)的方式来维护窗口内的最大值**。

这里使用**collections**里的**deque**，即**队列**，它可以**以o(1)的时间复杂度在头尾增删元素**。
deque维护窗口内的数据中的**有效数据（这里是nums的索引）**，有效是指对找最大值有用的数据。涉及3个过程：

- **增加数据**：窗口向前滑动，会带来新的元素，该元素必然要进入deque（为什么必然？因为它有可能是后续窗口的最大值）。那么在进入时可以对deque内已经存在的元素做一道过滤（去掉无效数据），如果deque末尾元素比新元素小（或相等），那么这个末尾元素在新元素到来后就没有存在的意义了，需要deque.pop()，直到deque末尾的元素比新元素大，才停止过滤，加入新元素；
  **tip：这样维护的deque的首元素就是最大值，也就是每个窗口的最大值答案**
- 删除数据：窗口向前滑动，那么有元素会被滑出去，则需要从deque内删除它。这里要分两种情况，被滑出去的数据可能是无效数据，在被滑出之前已经被pop掉了；另一种情况就是被滑出去的数据是有效数据，此时popleft，即删掉deque内的首个元素（为什么是删首位元素？因为deque在构建的时候，其顺序就是nums里从左到右的，只不过没有无效数据，相当于是窗口内的保持原nums相对顺序的有效数据）。
  这里具体的，就是**根据当前窗口右端所在索引和deque首元素（即此时窗口有效数据里的最大值的索引）来计算窗口长度，如果超出规定，则删首元素。**
- **记录答案**：当窗口左端是nums内的合法索引时，则记录deque的首元素索引对应数值。

```python
pre = 1
```

## 41.缺失的第一个正数

![image-20260305161820060](./images/image-20260305161820060.png)

只能用**o(1)的空间复杂度**，则需要**原地修改**数组，调整后的理想情况是：

```python
# nums的长度为n，其中nums[i]=i+1，即 索引i 处存放的 数字是i+1：
nums = [1,2,3,4,5,6,7]
index = [0,1,2,3,4,5,6]
```

按照这样排列后，**缺失的第一个正数就是n+1**

但情况绝不会如此简单：

**① 存在不在1-n之间的数字** ---> **不需安置它，它本身就没有自己的位置，最后分到哪就是哪**

```python
# 原始数组
nums = [3,4,-1,1]
# 往理想情况排列
# 3应该存放在索引3-1=2处，所以将索引0与索引2的元素交换
nums = [-1,4,3,1] 
# -1不在范围内，不管
# 下一个元素是4,4应该存放在索引4-1=3处，所以交换索引1与索引3的元素
nums = [-1,1,3,4]
# 此时在索引1的元素是1，它应该存放在索引1-1=0处，所以交换索引1与索引0的元素
nums = [1,-1,3,4]
# 此时所有元素已经安置好
# 缺失的第一个正数就是第一个 nums[i]!=i+1 的 i+1，这里就是2
```

**②数组内有重复元素** ---> **若它应该放的位置上已经有正确的数字，则不需要管它**

```python
# 原始数组
nums = [1,1,2,3]
# 第一个元素1应该放在索引1-1=0处，正确
# 第二个元素1应该放在索引1-1=0处，查看索引0发现已经放了正确的元素1，则此时第二个元素1暂不管
# 第三个元素2应该放在索引2-1=1处，故交换索引2与索引1的元素
nums = [1,2,1,3]
# 同理，不处理此时在索引2上的元素1
# 第四个元素3应该放在索引3-1=2处，故交换索引3与索引2的元素
nums = [1,2,3,1]
# 同样的，缺失的第一个正数就是第一个 nums[i]!=i+1 的 i+1, 这里是4
```

故**最终的处理逻辑**是：

遍历nums，给**nums[i]一个合理的去处**：

- 如果nums[i]不是1-n之间的数，则不需要管它

- 如果nums[i]是1-n之间的数，则它应该存放在索引nums[i]-1处：

  - 如果nums[i]-1就是i，则说明nums[i]放对了位置，不需要管它

  - 如果nums[i]-1不是i，说明nums[i]放错了位置，需要进一步看：

    - 若索引nums[i]-1处现有元素就是nums[i] (重复元素)，则不需要管nums[i]

    - 若索引nums[i]-1处现有元素不是nums[i] ，则交换索引为i和索引为nums[i]-1的元素

      这里**注意：先记录下需要交换的两个索引，避免因nums[i]改变而导致索引错误**

总结下来，需要交换索引的情况发生在：

`0=<nums[i]<=n and nums[nums[i]-1]!=nums[i]`

**所以唯一需要交换元素的情况是：**

nums[i]在1-n之间 **and** nums[i]-1!=i **and** nums[nums[i]-1]!=nums[i] 

( nums[i]-1!=i **and** nums[nums[i]-1]!=nums[i]  可以合并为 nums[nums[i]-1]!=nums[i] )

## 146.LRU缓存

![image-20260310200425779](./images/image-20260310200425779.png)

涉及的数据结构：

- 双向链表，加上dummy可以实现全连接（既可以看到链表的首元素，也可以看到链表的末尾元素）；
- 链表节点，需要有双向指针，存储的key和value；
- 字典dict，关键字key为键，存储的值为链表节点。

思想：

- 因为当容量达到设置值时，需要逐出最久未使用的元素，所以我们一旦使用了某元素（put/get），则需要把它调整到链表头(dummy后面)；
- 因为要以o(1)空间复杂度，所以引入字典dict来实现o(1)的查找；
- 超容量只会发生在put里，加入了新的，旧的才可能多余，所以在put之后要检查容量。

```python
class Node:
    def __init__(self, key=0, val=0, prev = None, next0 = None):
        self.key = key
        self.val = val
        self.prev = prev
        self.next = next0


class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.dummy = Node()
        self.dummy.prev = self.dummy
        self.dummy.next = self.dummy
        self.key2node = dict()

    # 获取key对应的节点，若没有则返回None；并将该节点移动链表头部
    def get_node(self, key: int) -> Optional[Node]:
        if key not in self.key2node:
            return None
        node = self.key2node[key]
        self.remove(node)
        self.push_front(node)
        return node
    
    def get(self, key: int) -> int:
        node = self.get_node(key)
        return node.val if node else -1

    def put(self, key: int, value: int) -> None:
        if key in self.key2node:
            node = self.get_node(key) # 此处在get_node已将使用过的置顶
            node.val = value
            return
        # 新增时，字典和链表要同步增加
        node = Node(key, value)
        self.key2node[key] = node
        self.push_front(node)
        if len(self.key2node) > self.capacity:
            denode = self.dummy.prev
            # 删除时，字典和链表要同步删除
            del self.key2node[denode.key]
            self.remove(denode)

    # 删除节点x 
    def remove(self, x: Node) -> None:
        x.prev.next = x.next
        x.next.prev = x.prev

    # 在链表头添加节点x
    def push_front(self, x: Node) -> None:
        x.prev = self.dummy
        x.next = self.dummy.next
        x.prev.next = x
        x.next.prev = x


# Your LRUCache object will be instantiated and called as such:
# obj = LRUCache(capacity)
# param_1 = obj.get(key)
# obj.put(key,value)
```

## 104.二叉树的最大深度

![image-20260310201642075](./images/image-20260310201642075.png)

### 方法1：递归

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        if not root:
            return 0
        if root.left is root.right:
            return 1
        
        return 1 + max(self.maxDepth(root.left), self.maxDepth(root.right))
```

### 方法2：迭代

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        
        def traverse(root: Optional[TreeNode], depth: int) -> None:
            if not root:
                return
            depth+=1
            nonlocal ans
            ans = max(ans, depth)
            traverse(root.left, depth)
            traverse(root.right, depth)

        ans = 0
        traverse(root, 0)
        return ans
```

```python
        return self.isSymmetric1(root.left,root.right)
```

## 102.二叉树的层序遍历

![image-20260311190817977](./images/image-20260311190817977.png)

直接看代码：

```python
from collections import deque

class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return [] # 一定要判断，否则把None加到队列里，并访问.val会报错
            
        ans = []
        q = deque()
        q.append(root)

        while q:
            sz = len(q)
            cur = []
            for _ in range(sz):
                node = q.popleft()
                cur.append(node.val)
                if node.left:
                    q.append(node.left)
                if node.right:
                    q.append(node.right)
            ans.append(cur.copy()) # 这里注意用copy

        return ans
```

## 437.路径总和3

## 236.二叉树的最近公共祖先

![image-20260311200557153](./images/image-20260311200557153.png)

![image-20260311200608474](./images/image-20260311200608474.png)

明确答案可能出现的方式：

- node为p（或q），q（或p）在node的左右子树中，则此时p（或q）为最近公共祖先；
- node不是p或q，p和q分别存在于node的左右子树里，则此时node为最近公共祖先。

遍历节点，三种情况：

- 节点为空，则返回None；

- 节点为p或q，则返回节点（因为它可能是最近公共祖先）；

- 节点非空，但又不是p或q，此时看它的左右子树递归调用的返回值：

  - 若左右子树的返回都是None，则返回None；

  - 若左右子树的返回都不是None（说明一个是p一个是q），则返回当前节点（最近公共祖先）;

  - 若左右子树的返回一个是None，一个不是None，则返回不是None的那个，因为它可能是p或q或最近公共祖先的节点。



## 124.二叉树中的最大路径和

![image-20260311202132626](./images/image-20260311202132626.png)

路径必有**拐点**，我们可以遍历每个节点，维护最大路径和ans，把该节点作为拐点的最大路径和与ans取max，最终ans为答案。

> [!NOTE]
>
> **如何计算某节点作为拐点的最大路径和？**
>
> 计算该节点的左子树向下延伸的最大路径和，右子树向下延伸的最大路径和，二者与当前节点的val相加，得到该节点作为拐点的最大路径和。
>
> 所以，用traverse来遍历每个节点，返回该节点向下延伸的最大路径和，如果该值小于0，则置0（即不经过这个节点了）。
